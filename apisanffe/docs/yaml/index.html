Fascicolo Finanziario Elettronico (FFE)

Avvertenze

  1. Questo YAML è stato pensato esclusivamente per il front-end destinato al cittadino
  2. Questo YAML è un contratto tra lo strato verde e lo strato rosso
  3. La tipologia del pagamento verrà calcolata nello strato rosso o verde
  4. La separazione tra "spese effettuate", "crediti" e "rimborsi" si fa usando il codice dello stato della pratica:
    • 801 -> rimborsati
    • 802 -> crediti
    • tutti gli altri -> spese effettute

Domande

  1. La funzionalità "Esporta lista pagamenti" chi la implementa?
  2. Sarebbe possibile avere tutti i codici possibili dei motivi di pagamento?

Up 
get /cittadini/{cit_id}/rimborsi
Lista rimborsi (cittadiniCitIdRimborsiGet)

Permette di leggere l'elenco dei rimborsi di un cittadino.

E' inoltre possibile filtrare l'elenco secondo diverse condizioni descritte nei query params

Path parameters

cit_id (required)
Path Parameter — Il codice fiscale del cittadino su cui operare

Request headers

Query parameters

data_dal (optional)
Query Parameter — Data di inizio del periodo entro cui cercare le pratiche di rimborso
data_al (optional)
Query Parameter — Data di fine del periodo entro cui cercare le pratiche di rimborso
stato_ticket (optional)
Query Parameter —

Permette di filtrare per prendere solo i ticket in un certo stato. I valori ammessi sono:

  • 801 - Rimborsato
  • 802 - Da rimborsare
in_erogazione (optional)
Query Parameter —

Permette di filtrare in base allo stato di erogazione del rimborso. I valori ammessi sono:

  • 0 - Non erogato
  • 1 - In erogazione (credito incluso in un pagamento tramite borsellino)
  • 2 - Erogato (il voucher o il bonifico è stato erogato)
modalita_rimborso (optional)
Query Parameter —

Permette di filtrare in base alla modalità di rimborso. I valori ammessi sono:

  • 1 - Voucher
  • 2 - Bonifico
  • 3 - Borsellino elettronico
id_asr (optional)
Query Parameter — Permette di filtrare per ASR.
motivo_pagamento (optional)
Query Parameter — Motivo pagamento del pagamento oggetto di rimborso.
codice_versamento (optional)
Query Parameter — Codice versamento del pagamento oggetto di rimborso.
codice_servizio (optional)
Query Parameter — Identificativo del sistema dipartimentale che ha emesso il pagamento oggetto di rimborso.

Return type

array[model_lista_ticket]
